# cullich
A small web app to allow you to quickly cull your immich photos by month/yeah.

## Introduction
My photo library is full of photos I never intended to keep but made to quickly share to someone. Going over this in Immich (or the built-in photo library on my phone) manually is annoying and as a result I have not been keeping up with this. So I envisioned a tool that allowed me to quickly cull my photos into keepers and rejects which resulted in this app, cullich.

![Screenshot of Cullich](https://github.com/maxvw/cullich/raw/main/docs/screenshot.png)

Most of the code in this repo is generated by Claude as an experiment.

> **NOTE:** I don't expect to keep working on this project so it is provided as-is, but open to contributions.

## Important Note
The first thing to note is that cullich does not perform destructive actions, it will only show your photos/videos and tag/untag as needed. To actually cleanup (archive or delete) you need to navigate to Immich, filter by tag and perform whatever action you desire.

## Usage
When loading cullich in the web browser it will default to the current time bucket, meaning the current month+year. It will show photos and videos in a timeline with one highlighted.

To navigate between them you can use the arrow keys, however ideally you immediately start culling with the `,` (reject) and `.` (keep) buttons. On doing this it will mark it as reject or keep and advance to the next slide. Spacebar "zooms" for closer inspection, or plays video. Nothing is actually persisted until you hit the Save button. In the top left you can switch to another month/year.

## Keyboard Shortcuts
| Shortcut | Description |
| --- | --- |
| Arrow Left | Previous photo/video |
| Arrow Right | Next photo/video |
| CMD+S  | Show save dialog (Enter submits, Esc cancels) |
| CMD+Z | Undo last action |
| CMD+SHIFT+Z | Redo last action |
| , | Reject current photo/video |
| . | Keep current photo/video |
| Space | Zoom or play video |


## Environment Variables
Here is a quick overview (also available in `.env.example`).

| Name | Required | Description |
| --- | --- | --- |
| IMMICH_BASE_URL | yes | e.g. https://immich.example.com/api |
| IMMICH_API_KEY | yes | [Create an API key here](https://my.immich.app/user-settings?isOpen=api-keys) |
| IMMICH_TAG_KEEP | no | Defaults to "Cullich-Keep" |
| IMMICH_TAG_REJECT | no | Defaults to "Cullich-Reject" |

## Development
To install dependencies:

```bash
bun install
```

To start a development server:

```bash
bun dev
```

This project was created using `bun init` in bun v1.2.16. [Bun](https://bun.sh) is a fast all-in-one JavaScript runtime.
